================================================================================
  1. PROJECT OVERVIEW
================================================================================

JCdocs is a fully offline, premium Android office suite built with Kotlin and
Jetpack Compose. It features:

  - Word Document Editor (rich text formatting, themes, margins, columns, PDF export)
  - Spreadsheet Editor (dynamic formulas: SUM, AVG, arithmetic, cell references)
  - Slide Presentation Editor (multi-slide with themes, full-screen present mode)
  - AI Conversational Assistant (multi-provider: Gemini, OpenRouter, Custom)
  - File management (search, filter by type, favorites, CRUD)

Styled after ONLYOFFICE (terracotta/orange branding). Designed for API 24-36.

  4.4  AI Provider Layer
--------------------------------------------------------------------

  AiProvider.kt (interface):
    - suspend fun chat(messages, systemPrompt): Result<AiResponse>
    - suspend fun testConnection(): Boolean

  GeminiAiProvider.kt:
    - Model: gemini-3.5-flash via generativelanguage.googleapis.com
    - API: POST /v1beta/models/gemini-3.5-flash:generateContent?key=API_KEY
    - GeminiRequest/Response DTOs with Moshi @JsonClass
    - 30s connect / 60s read timeouts

  OpenRouterAiProvider.kt:
    - 12+ free models with automatic fallback chain
    - Models: gemini-2.5-flash, deepseek-r1, gpt-oss-20b, llama-3.3, qwen-2.5, phi-3, etc.
    - Headers: Authorization (Bearer), HTTP-Referer, X-Title
    - Falls through models on failure; breaks on 401/403
    - Key preview in error messages (first 5 + last 5 chars)

  CustomAiProvider.kt:
    - OpenAI-compatible API (reuses OpenRouterApi interface)
    - User-configurable: baseUrl, apiKey, model
    - Default: https://api.openai.com/v1/ with gpt-3.5-turbo

    - shiftSpans: adjusts indices when text is inserted/deleted

    - WorkspaceMenuBar has Undo/Redo buttons

  PDF Export:
    - Uses Android's PdfDocument API
    - ExportButton triggers CreateDocument SAF launcher
    - Page dimensions from pageFormat/customDimensions
    - Writes draftContent as simple text on canvas

  7. DATA FLOW DIAGRAMS
================================================================================

--------------------------------------------------------------------
  7.1  Document Edit Flow
--------------------------------------------------------------------

  User types in editor
       │
       ▼
  onContentChange(newText) ──→ DocViewModel.updateDraftContent()
       │                            │
       │                            ▼
       │                        _draftContent.value = newText
       │                            │
       │                            ▼
       │                        saveChangesLocally()
       │                            │
       │                            ▼
       │                        repository.insertDocument(updated)
       │                            │
       │                            ▼
       │                        Room DB (documents table) updated
       │
       ▼
  LaunchedEffect(draftContent) syncs TextFieldValue
       │
       ▼
  UI recomposes with new TextFieldValue

  Note: Every keystroke triggers DB write via saveChangesLocally().
  This means auto-save is real-time with no debouncing.

--------------------------------------------------------------------
  7.2  Spreadsheet Formula Evaluation Flow
--------------------------------------------------------------------

  User edits cell "C2" with "=A2*B2"
       │
       ▼
  DocViewModel.updateCellExpression("C2", "=A2*B2")
       │
       ▼
  _sheetData updated (Map<String, String>)
       │
       ▼
  UI recomposes with new data
       │
       ▼
  getCellValue("C2") called for display
       │
       ▼
  evaluateCell("C2", sheetData, visited=[])
       │
       ├── expression = "=A2*B2"
       ├── Not SUM, not AVG
       ├── Math operator: *
       │     ├── evaluateCell("A2") → "120"
       │     ├── evaluateCell("B2") → "3"
       │     └── 120.0 * 3.0 = 360.0
       └── Returns "360"
       │
       ▼
  Cell displays "360"

  Supported formulas:
    - =SUM(range)   e.g. =SUM(A1:C3)
    - =AVG(range)   e.g. =AVG(B2:B10)
    - Arithmetic: =A1*B2, =A1+B2, =A1-B2, =A1/B2
    - Simple reference: =A1 (copies value)
    - Circular ref detection → "#REF!"

--------------------------------------------------------------------
  7.3  AI Chat Message Flow
--------------------------------------------------------------------

  User taps send
       │
       ▼
  DocViewModel.sendMessage(text)
       │
       ├── If no active conversation → createConversation("New Chat")
       │
       ▼
  performMessageSend(conv, text)
       │
       ▼
  1. Save user message to Room (aiRepository.saveMessage)
  2. Load message history (chatDao.getMessagesList)
  3. Map to AiMessage list
  4. Get active provider (aiRepository.getActiveProvider)
       │
       ▼
  provider.chat(history)
       │
       ├── GeminiAiProvider → Gemini API HTTP call
       ├── OpenRouterAiProvider → OpenRouter API (model fallback chain)
       └── CustomAiProvider → OpenAI-compatible API
       │
       ▼
  onSuccess:
    - Save AI response message to Room
    - Auto-rename conversation if title was "New Chat"
    - Clear any error state
  onFailure:
    - Set _aiError.value with error message
    - If OpenRouter auth error, show hint about missing key

--------------------------------------------------------------------
  7.4  Undo/Redo Flow
--------------------------------------------------------------------

  User makes edit
       │
       ▼
  pushSnapshot() called
       │
       ▼
  undoRedoManager.pushState(captureSnapshot())
       │
       ├── Captures: title, content, TextFieldValue, spans,
       │   editorTheme, margins, columns, alignment, fontSize,
       │   landscape, pageNumber position/format/startAt
       ├── Deep copies spans list
       ├── Deduplicates consecutive identical states
       └── Max 500 states
       │
       ▼
  User taps Undo
       │
       ▼
  undoRedoManager.undo(currentSnapshot)
       │
       ├── Push current state to redoStack
       └── Pop from undoStack
       │
       ▼
  Restore all state fields
  DocFormatRepository.getSpans().clear() + addAll(restored)
  UI recomposes

  9.2  GeminiAiProvider
--------------------------------------------------------------------

  Uses: Google Gemini API (generativelanguage.googleapis.com)
  Model: gemini-3.5-flash
  Auth: API key via query parameter ?key=API_KEY
  API: POST /v1beta/models/gemini-3.5-flash:generateContent

  Request format:
    {
      "contents": [{ "role": "user"/"model", "parts": [{ "text": "..." }] }],
      "systemInstruction": { "parts": [{ "text": "..." }] }  (optional)
    }

  Default key source: BuildConfig.GEMINI_API_KEY (from .env secrets)
  Timeout: connect 30s, read 60s, write 30s

--------------------------------------------------------------------
  9.3  OpenRouterAiProvider
--------------------------------------------------------------------

  Uses: OpenRouter API (openrouter.ai/api/v1)
  Auth: Bearer token via Authorization header
  Headers: HTTP-Referer, X-Title for rankings

  Model fallback chain (12 free models):
    1. google/gemini-2.5-flash:free
    2. google/gemini-2.5-pro:free
    3. deepseek/deepseek-r1:free
    4. openai/gpt-oss-20b:free
    5. deepseek/deepseek-chat:free
    6. meta-llama/llama-3.3-70b-instruct:free
    7. meta-llama/llama-3.1-8b-instruct:free
    8. qwen/qwen-2.5-72b-instruct:free
    9. qwen/qwen-2.5-coder-32b-instruct:free
   10. mistralai/mistral-7b-instruct:free
   11. microsoft/phi-3-medium-128k-instruct:free
   12. openrouter/auto (last resort)

  Breaks on 401/403 (auth error — no point retrying)
  Error messages include key preview for debugging

  9.4  CustomAiProvider

  10. STATE MANAGEMENT

--------------------------------------------------------------------
  10.1 ViewModel StateFlows
--------------------------------------------------------------------

  DocViewModel uses MutableStateFlow for mutable state and
  exposes as StateFlow for the UI layer.

  Derived/computed flows:
    - documents = combine(allDocuments, searchQuery, selectedTypeFilter)
    - aiMessages = activeConversation.flatMapLatest { getMessages(it.id) }

  All state reads use collectAsStateWithLifecycle() in composables.

--------------------------------------------------------------------
  10.2 Compose State Hoisting Pattern
--------------------------------------------------------------------

  Each composable receives state as parameters and callbacks as lambdas.
  Example SidebarExplorer signature:
    (documents, selectedDoc, draftTitle, searchQuery, selectedFilter,
     onSearchChange, onFilterChange, onDocSelect, onDocDelete,
     onDocFavoriteToggle, onCreateClick, ...)

  WorkspacePane has extensive local state (mutableStateOf):
    editorTheme, pageMargins, columnCount, textAlignment, fontSize,
    isLandscape, editorTextFieldValue, activeRibbonTab, etc.

--------------------------------------------------------------------
  10.3 DocFormatSpan System
--------------------------------------------------------------------

  DocFormatSpan(start, end, type, value) — immutable-ish data class
  DocFormatRepository — singleton object with per-docId span storage

  Key operations:
    - getSpans(docId) → SnapshotStateList<DocFormatSpan>
    - applySpan(docId, type, value, start, end) — merges overlapping
    - removeSpanRange(docId, start, end) — removes/truncates
    - removeSpanTypeRange(docId, type, start, end) — type-specific removal
    - hasSpan(docId, type, start, end) — check if selection fully covered
    - shiftSpans(docId, changeStart, deleted, inserted) — adjust on text edit

  Formatting types: "bold", "italic", "underline", "strikethrough",
                    "subscript", "superscript", "color", "highlight"
